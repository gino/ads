We have to restructure our launch ads flow (https://chatgpt.com/c/68e165a6-ebcc-832c-8bb1-154f77b8d92b).

- For image ads:
Frontend sends the file (image), name, adSetId, facebookPageId, etc.
Backend uploads this image to Meta to their /adimages endpoint which returns a `hash`
We send this hash to the frontend

On the frontend we loop through all the hashes that we 'received' and then we hit our /upload/create-ad endpoint which creates the ad creative AND ad.

- For video ads:
Frontend sends the video file, a thumbnail (base64 → converted to File), name, adSetId, facebookPageId, etc.
Backend first uploads the thumbnail image to Meta’s /adimages endpoint to get an image_hash.
Then it uploads the video file to Meta’s /advideos endpoint, which returns a video_id.
We send both the video_id and image_hash back to the frontend.
On the frontend, we loop through all the uploaded videos and make a request to /upload/create-ads, which creates both the ad creative and the ad, using the video_id and image_hash.
